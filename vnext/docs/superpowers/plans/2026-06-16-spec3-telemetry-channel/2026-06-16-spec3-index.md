# Spec 3 — Telemetry Channel in ExecuteResult: Implementation Plan Index

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Move telemetry off the Spec-2 stream-decorator into a typed channel on `ExecuteResult`, so every endpoint (chat-completions, messages, responses, gemini) records usage + performance through one shared `respond-telemetry` path and the legacy `dispatch.ts` + `conversation-attempt.ts` + `usage-tracker.ts` triple can be deleted.

**Architecture:** `EventResult.modelIdentity` becomes required; `EventResult.performance` + `*ErrorResult.performance` become optional; `EventResult.finalMetadata?: Promise<EventResultMetadata>` carries replacement metadata for interceptors that substitute the stream. A new `data-plane/chat-flow/shared/telemetry-ctx.ts` exports `TelemetryRequestContext` (separate from `@vnext/interceptor`'s minimal `RequestContext`). Persistence calls wrap I/O in `@vnext/platform.waitUntil`.

**Tech Stack:** Bun + TypeScript, Hono (data-plane HTTP), `@vnext/protocols/common` (types), `@vnext/platform` (background executor), existing `Repo` (no signature changes).

## Plan parts

1. **[Part 1 — Shared helpers](./part-1-shared-helpers.md)** — Protocols types extension (`TelemetryModelIdentity`, `PerformanceTelemetryContext`, `EventResultMetadata`, `EventResult` extension, `eventResult` factory). New shared modules: `attempt-helpers.ts`, `respond-telemetry.ts`, `telemetry-ctx.ts`. Rewrite `upstream-telemetry.ts` to a pure terminal-frame classifier (drop the recorder interface). No endpoint migration yet — old `withUpstreamTelemetry` recorder stubs in attempt.ts stay alive on a temporary call site.

2. **[Part 2 — chat-completions migration](./part-2-chat-completions.md)** — Wire chat-completions `attempt.ts` + `respond.ts` to the new helpers. `serve.ts` constructs `TelemetryRequestContext` and passes it through. Delete the recorder-based `withUpstreamTelemetry` call. Add the one-endpoint acceptance battery (usage row written, performance row written for success / upstream-error / internal-error, modelKey corrected from upstream).

3. **[Part 3 — messages + responses migration](./part-3-messages-responses.md)** — Migrate `messages` and `responses` off `dispatch()`. Both endpoints get their own `attempt.ts` + `respond.ts`. `responses/snapshot-sidecar` and `responses/image-generation-shortcut` fill `finalMetadata` for interceptor-replaced streams.

4. **[Part 4 — gemini + cleanup](./part-4-gemini-and-cleanup.md)** — Migrate `gemini` (chat verb only — count-tokens stays out of scope per spec §3). Delete `dispatch.ts`, `conversation-attempt.ts`, `usage-tracker.ts`, and their test files. Migrate the 6 preexisting failing dispatch-related tests onto the new chain. Run SDK regression suite.

## Cross-part invariants

- **Repo signatures unchanged.** All four parts only relocate callers of `repo.usage.record` / `repo.apiKeys.touchLastUsed`.
- **`ProviderResponse` unchanged.** `modelKey` correction is observed lazily by `SourceStreamState` from the first frame's `model` field.
- **`@vnext/platform.waitUntil` is the only background mechanism.** No new context fields.
- **Test discipline.** Every new module gets a unit test in the same task. Every endpoint migration gets an e2e test mirroring the Spec-2 `include-usage-wiring.test.ts` pattern (real Hono app, stubbed repo, captured fetch).
