# Spec 6: Cross-Protocol Attempt Wiring — Implementation Plan Index

> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:subagent-driven-development` (recommended) or `superpowers:executing-plans` to implement this plan part-by-part. Steps use checkbox (`- [ ]`) syntax for tracking.

**Spec:** `vnext/docs/superpowers/specs/2026-06-17-spec6-cross-protocol-attempt-wiring.md`

**Goal:** Remove the three 501 "cross-protocol attempts not yet supported" short-circuits in `chat-completions/attempt.ts`, `messages/attempt.ts`, `responses/attempt.ts`, and unify all four data-plane attempts to route cross-protocol requests through a shared `traverseTranslation` helper. Restore parity with the reference `copilot-gateway` implementation.

**Architecture:** New `traverseTranslation` helper that calls `translator.translateRequest`, recursively invokes the hub protocol's `attempt.generate`, and wraps the returned events with `translator.translateEvents`. Each source `attempt.ts` dispatches to the helper when `sel.targetEndpoint !== ownProtocol`. `respond.ts` non-streaming branch consults `result.translateBody` to map hub JSON → source JSON without re-streaming.

**Tech Stack:** TypeScript, Bun runtime, Hono, ESM cyclic imports between attempt modules (already used today).

## Why 4 parts

The spec touches 4 protocols + protocols-package types + translate package. A single mega-PR would be hard to review and risky to land. Splits track natural risk gates from spec §6 acceptance:

| Part | Risk gate | Lands when |
|---|---|---|
| **Part 1: Foundation** | Types + helpers compile, no attempt rewired yet | `bun typecheck` clean; `traverseTranslation` unit tests green; no behavior change |
| **Part 2: chat-completions wiring** | First attempt cuts over; pattern proven | Acceptance §6.2 cases 1 & 3 pass (cc → responses, cc → messages) |
| **Part 3: messages + responses wiring** | Two more attempts cut over | Acceptance §6.2 case 2 passes (responses → chat_completions); messages cross-protocol works |
| **Part 4: gemini refactor + integration battery + SDK regression** | All four attempts unified; static gates green | Acceptance §6.2 case 4 passes; §6.3 SDK regression no new failures; §6.4 grep confirms no 501 strings |

Each part ends with a commit and a passing test gate. Parts 2-4 each depend on Part 1 but are independent of each other (the four `attempt.ts` files don't import each other directly — calls go through `pickHubAttempt` which is constructed in Part 1).
